
Архитектура проекта

App - главный класс приложения; он инициализирует работу сервиса. Включает сессию, создает контейнер свойств и объекты
Request и Response. При этом контейнер свойств реализует шаблон сингтона (поскольку это общие для всего приложения свойства).

Router - класс маршрутизатор. Его задача - парсинг запросов и подключение нужных контроллеров;

В состав App входит контейнер свойств приложения (smpt порты, логин, пароль и т.д.)
В состав App входит класс Request. Его задача - работа с входными данными запроса.
В состав App входит класс Response. Его задача - формирование ответа сервиса.

Также существует два абстрактных класса - Controller и Model.

Первый - надмножество для всех прочих контроллеров.
В нем реализуются базовые для всякого контроллера методы (в том числе взаимодействие с моделями/сущностями).

Второй - надмножество для всех прочих моделей данных, иначе говоря -
сущностей, которыми оперирует сервис.
